a:T8f1,<p>I started this as a performance task: downscale assets, add <code>srcSet</code>/<code>sizes</code>, and improve LCP. The bigger problem turned out to be maintainability.</p>
<p>Image behavior was spread across scripts and components with repeated conventions:</p>
<ul>
<li>variant naming assumptions,</li>
<li>hardcoded <code>srcSet</code> strings,</li>
<li>multiple script aliases and legacy paths.</li>
</ul>
<p>That made drift easy, and small changes in one place could quietly break expectations elsewhere. I replaced that with a simpler pattern: generate one manifest at build time (<code>src/generated/imageVariantManifest.json</code>) and resolve variants from it at runtime.</p>
<h2>What got simpler</h2>
<p>Three things became simpler:</p>
<ol>
<li>One canonical workflow:
<ul>
<li><code>yarn image:variants</code></li>
</ul>
</li>
<li>Less duplicated rendering logic:
<ul>
<li>shared <code>ResponsiveImage</code> component instead of repeated <code>&#x3C;picture></code> patterns</li>
</ul>
</li>
<li>Static assets became data-driven:
<ul>
<li>background and portrait <code>srcSet</code> values now come from manifest-backed metadata, not JSX literals</li>
</ul>
</li>
</ol>
<h2>Human constraint: it should just work</h2>
<p>This was an important constraint for me: I do not want to manually create <code>-sm</code>, <code>-md</code>, <code>-lg</code> files every time I add an image.</p>
<p>The workflow is now:</p>
<ol>
<li>Add the source image.</li>
<li>Reference it in frontmatter or markdown.</li>
<li>Run <code>yarn image:variants</code>.</li>
</ol>
<p>That keeps authoring simple while still making outputs consistent.</p>
<h2>Reliability and Performance as Outcomes</h2>
<p>Once maintainability improved, reliability and performance improved with it. I also removed silent fallback for required profiles (<code>cover.card</code>, <code>cover.hero</code>, <code>inlineContent</code>). If a required variant is missing, it now fails fast instead of shipping a hidden regression.</p>
<h2>Takeaway</h2>
<p>I started by chasing LCP. The durable fix was maintainability: fewer scattered conventions, one source of truth, and clearer build/runtime boundaries.
