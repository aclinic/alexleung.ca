8:Td05,<p>I recently finished implementing the blog section of this personal website. Rather than reaching for a complex CMS, I decided to build it directly into my existing Next.js application using standard tools.</p>
<h2>Low-Maintenance Publishing</h2>
<p>I wanted a place to share technical thoughts without adding maintenance overhead. The requirements were simple:</p>
<ol>
<li><strong>Write in Markdown</strong>: I want to write posts in VS Code.</li>
<li><strong>Zero Runtime Cost</strong>: The blog should be statically generated.</li>
<li><strong>Fast</strong>: It should feel instant.</li>
</ol>
<h2>Static by Default</h2>
<h3>1. Filesystem as the Database</h3>
<p>I created a simple utility, <code>blogApi.ts</code>, that reads directly from the file system. It uses <code>gray-matter</code> to parse the frontmatter and a <code>remark</code>/<code>rehype</code> pipeline—specifically <code>rehype-pretty-code</code>—to process the content.</p>
<h3>2. Static Routing</h3>
<p>Next.js makes it straightforward to turn a folder of markdown files into routes. I used <code>generateStaticParams</code> to tell Next.js which paths to build at compile time:</p>

<p>This ensures that <code>alexleung.ca/blog/boring-blog-architecture</code> is just a simple HTML file on the server, not a dynamic Node.js request.</p>
<h3>3. Details That Mattered</h3>
<p>To make it feel professional, I spent time on the details:</p>
<ul>
<li><strong>Typography</strong>: I used <code>@tailwindcss/typography</code> but customized it to remove the default backticks from inline code for a cleaner look.</li>
<li><strong>Metadata</strong>: Each post automatically generates its own SEO tags and JSON-LD structured data.</li>
<li><strong>Syntax Highlighting</strong>: I chose <code>rehype-pretty-code</code> (powered by Shiki). It uses the same TextMate grammars as VS Code, meaning the highlighting is extremely accurate. Because it generates inline styles, there's no brittle CSS to import from <code>node_modules</code>.</li>
<li><strong>Sitemap</strong>: A dynamic script crawls my posts directory to keep <code>sitemap.xml</code> up to date automatically.</li>
</ul>
